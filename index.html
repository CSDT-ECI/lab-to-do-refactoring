About the Project

From technical debt
to clean architecture

LabToDo is a task management system built for the computer science laboratory at Universidad Escuela Colombiana de Ingeniería Julio Garavito. It manages tasks, user assignments, and semester tracking with a role-based authentication system (Administrator, Monitor, Student).

This refactoring initiative applied seven CSDT_M disciplines to measure and progressively reduce the debt accumulated in the original codebase — starting from a system with no tests, 89 code smells, and a 527-line controller mixing 15+ unrelated responsibilities.

  • → Test coverage raised from 0% to 88%, surpassing the 85% JaCoCo quality gate
  • → 6-stage DevSecOps pipeline integrating OWASP, Semgrep SAST, and SonarCloud — runs complete in under 5 minutes
  • → 7 architectural smells mapped against the Sharma taxonomy and ISO/IEC 25010 quality model
  • → Spec-Driven Development applied via AIDLC guardrails as an AI-assisted quality gate

Testing Debt & Code Quality

DevSecOps / CI-CD

Designed a 6-stage diamond pipeline: Build → {Test · SonarCloud · OWASP · Semgrep} in parallel → Deploy. Security findings surface at development time, not deployment time. Concurrency control cancels stale runs automatically.

Applied Spec-Driven Development (SDD) from the AIDLC framework — the CLAUDE.md file as a mega-prompt defining context, constraints, acceptance criteria, and edge cases. AI red teaming used as a quality gate after each generation step.
